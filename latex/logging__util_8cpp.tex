\doxysubsection{Detailed Description}
Utilities for logging data. 

\hypertarget{logging__util_8cpp_autotoc_md2}{}\doxysubsection{\texorpdfstring{Summary}{Summary}}\label{logging__util_8cpp_autotoc_md2}
Define the helper functions/utilities which have been written for logging data on the Data Gators.

Emphasis is placed on making this as simple to use as possible. Unfortunately it is likely that the implementation is rather difficult to follow due to the number of edge cases covered. These include\+:


\begin{DoxyItemize}
\item multiple logging \doxylink{src_2main_8cpp_a2df27a00b191de3537c3b53adcecf1e8}{interfaces(\+SD, Wi\+Fi, etc)}
\item cases where network connection is not available to obtain timestamp for data
\end{DoxyItemize}\hypertarget{logging__util_8cpp_autotoc_md3}{}\doxysubsubsubsection{\texorpdfstring{Key Functionality}{Key Functionality}}\label{logging__util_8cpp_autotoc_md3}
Key functionality includes functions for\+:


\begin{DoxyEnumerate}
\item caching and retreiving data from NVS
\begin{DoxyItemize}
\item timestamps
\item log offsets (calculated from \# of resets)
\end{DoxyItemize}
\item constructing file names for logging
\item logging to SD card file
\end{DoxyEnumerate}

The timestamp, last exact timestamp + offset to write to non-\/volatile storage. This timestamp will be retrieved on next reset and replaced if network connection to NTP available. If NTP not available, then the offset is incremented.
