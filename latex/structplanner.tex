\doxysubsection{Detailed Description}
Data structure for task scheduling stored in NVS. 

Counters for when the last time the task was executed, the last reset counter value when reading was taken. Members are named by task name and the {\ttfamily \+\_\+t0} denotes physics-\/esque mathematical notation, ie this is the starting point of our calculation.

Tasks are similarly named functions which are run when the current reset count value minus the previous reset count recorded in {\ttfamily \texorpdfstring{$<$}{<}task\+\_\+name\texorpdfstring{$>$}{>}\+\_\+t0} equals the time between task executions specified in the config header.
